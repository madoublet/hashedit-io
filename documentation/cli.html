Command Line Interface (CLI)

Hashedit has an easy-to-use CLI to speed setup and adding users.

Setup

Initial setup that creates the configuration files and adds a user. Adding a user is optional if you intend to use third party authentication.

> node hashedit -s http://mysite.com -e sample@email.com -p mypassword
Option Format
-s [url] Runs the setup: (1) creates the ./config.js and ./data/users.json files (2) copies the URL into ./config.s
-e [email] The email used to login.
-p [password] The password used to login (stored as bcrypt hash)

Add Local User

Adds a local user. A local user's credentials are stored on the server. He/she logs in with a username and password (versus Google or other third party).

> node hashedit -e sample@email.com -p mypassword
Option Format
-e [email] The email used to login.
-p [password] The password used to login (stored as bcrypt hash)
