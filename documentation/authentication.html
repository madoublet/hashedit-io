Authentication

Learn how to setup authentication for Hashedit.

Google

1. Login to the Google Developer Console

2. Create a Project

Select dropdown from header and click "Create a project...". Enter a name for your project (e.g. Hashedit Site).

3. Enable and Manage APIS

Found in the first card under "Use Google APIs".

4. Get Credentials

This is in the left menu. Select "Add Credentials". Then select "OAuth 2.0 Client ID". And, configure the consent screen. You should then be given the Client ID and Client Secret. Put these in your config.js file.

5. Enable Google+ API

Go back to your Overview and search for Google+ and then enable that API.

6. Add Authorized Redirect URI

Go back to Credentials. Select the Client ID you created. Add an Authorized Redirect URI and a Authorized JavaScript origin. See the table below for the format.

Setting Format
Authorized Redirect URI http://my-hashedit-site.io/auth/google/callback
Authorized JavaScript Origin http://my-hashedit-site.io

7. Restart your application (not necessary if you are using nodemon)

It might take a few minutes to get it work.
