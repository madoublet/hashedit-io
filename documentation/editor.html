Setting up the Editor

The editor is built as a JavaScript singleton. You create the editor by running hashedit.setup as shown below.

// setup hashedit
hashedit.setup({
    path: '/node_modules/hashedit/',
    stylesheet: '/node_modules/hashedit/dist/hashedit-min.css',
    sortable: ['.sortable, .column, .carousel-caption']
});

Parameters for setup are shown in the table below.

Parameter Description
path Required. The path (absolute or relative URL) to the editor.
stylesheet Required. The path (absolute or relative URL) to the hashedit stylesheet.
sortable Required. An array of selectors to the containers that are sortable. These containers must be contained in a [hashedit] element.
demo Optional. (true | false) Whether to show the editor in demo mode. Authentication is bypassed, but you cannot save from Demo mode.

Hashedit requires the following hierarchy for sorting to work. Sortable items (e.g. shown as the H1 and P tag below) must be contained in a "sortable" container as specified in the sortable parameter. The sortable container must be descendent of a container with a hashedit and hasehdit-selector attribute.

<div id="my-container" class="container" hashedit hashedit-selector="#my-container">
    <div class="sortable">
        <h1>Sample Paragraph</h1>
        <p>Sample Paragraph</p>
    </div>
</div>

When you click save in the editor, an array is sent back to the server with a selector and updated HTML for each [hashedit] element.
